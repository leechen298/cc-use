# cc-use 0.4.0

> Auto profile routing, usability checks, and `cc-use status`

## Summary

`0.4.0` adds preflight auto profile routing. `cc-use` can now check configured profiles before launch, select the first usable profile, and then start Claude Code with the same isolated/shared launch semantics as before.

This release keeps `cc-use` as a launcher. It does not add mid-run failover, provider quality scoring, or a billing dashboard.

## Highlights

### 1. `cc-use auto`

New automatic isolated launch:

```bash
cc-use auto
cc-use auto -p "review this diff"
```

What it does:

- resolves the configured default profile first
- checks fallback profiles in order
- selects the first profile with `usable === true`
- launches the selected profile in isolated mode under `~/.cc-use/sessions/<profile>/`

### 2. `cc-use with auto`

New automatic shared-context launch:

```bash
cc-use with auto
```

What it does:

- uses the same auto-routing selection path
- launches the selected profile with native `~/.claude/` as `CLAUDE_CONFIG_DIR`
- keeps the `with` mode behavior explicit

### 3. `cc-use status`

New status view:

```bash
cc-use status
```

It prints the last known usability state for auto-participating profiles and marks stale cache entries.

The cache lives at:

```text
~/.cc-use/status.json
```

This file is a cache only. It is sanitized and must not contain API keys, raw provider responses, or full account payloads.

## Auto-routing config

Auto routing is configured in `~/.cc-use/config.json`:

```json
{
  "default": "deepseek",
  "auto": {
    "cacheTtlSeconds": 60,
    "fallbackOrder": ["deepseek", "kimi", "mimo-plan"],
    "profiles": {
      "deepseek": {
        "mode": "payg",
        "check": { "kind": "probe" }
      },
      "kimi": {
        "mode": "payg",
        "check": { "kind": "probe" }
      },
      "mimo-plan": {
        "mode": "token_plan",
        "check": { "kind": "manual_availability", "available": true }
      }
    }
  }
}
```

Supported check kinds:

- `probe`: sends a minimal Anthropic-compatible Messages request with `max_tokens: 1`
- `manual_availability`: uses an explicit configured boolean
- `api`: reserved balance-adapter path; no concrete adapter is bundled yet

Unsupported balance adapters fail closed as `check_failed`.

## Current command semantics

`0.4.0` intentionally preserves the existing launch semantics:

```bash
cc-use auto
```

- auto-select usable profile
- launch isolated

```bash
cc-use with auto
```

- auto-select usable profile
- launch shared native `~/.claude/`

```bash
cc-use <profile>
```

- launch named profile isolated

```bash
cc-use with <profile>
```

- launch named profile shared

```bash
cc-use isolate <profile>
```

- launch named profile isolated

## Boundaries

Not included in this release:

- real provider balance adapters
- usage ledger
- before/after balance delta recording
- config editing commands for auto routing
- mid-run provider switching
- task/model quality routing
- changing the default command model to shared/with-first mode

`recordUsage` is parsed for forward compatibility only. No usage ledger is written in `0.4.0`.

## Upgrade notes

No breaking change is introduced for existing workflows:

- `cc-use <profile>` still behaves as isolated mode
- `cc-use` with a configured default profile still behaves as isolated mode
- `cc-use with <profile>` still behaves as shared native `~/.claude/` mode

Auto routing is opt-in through:

```bash
cc-use auto
cc-use with auto
```
