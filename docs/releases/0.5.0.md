# cc-use 0.5.0

> With-first default semantics for Claude Code profile launches

## Summary

`0.5.0` changes the default launch model from isolated-first to with-first.

By default, `cc-use` now reuses the native Claude Code context under `~/.claude/` and only swaps provider/profile environment variables for the launched child process. Explicit isolated sessions remain available through `cc-use isolate`.

This is a default-behavior change, so users who rely on the old per-profile `CLAUDE_CONFIG_DIR` should move those commands to `cc-use isolate <profile>`.

## Highlights

### 1. Bare profile launches are now shared

```bash
cc-use deepseek
cc-use deepseek -p "review this diff"
```

These now launch the selected profile with native `~/.claude/` as `CLAUDE_CONFIG_DIR`.

Before `0.5.0`, bare profile launch used:

```text
~/.cc-use/sessions/<profile>/
```

Since `0.5.0`, use explicit isolated mode for that behavior:

```bash
cc-use isolate deepseek
```

### 2. Default-profile launch is now shared

```bash
cc-use
cc-use -p "review this diff"
cc-use -- -p "review this diff"
```

These resolve the configured default profile and launch with native `~/.claude/`.

If no default profile is configured in non-interactive mode, `cc-use` prints a clear error and exits:

```text
cc-use: no default profile set. Run 'cc-use default <profile>' or 'cc-use init'.
```

### 3. Auto routing now launches shared by default

```bash
cc-use auto
cc-use auto -p "review this diff"
cc-use auto -- -p "review this diff"
```

Auto routing still means:

```text
select the first usable profile before launch
```

The checker/router model from `0.4.0` is unchanged. The launch mode after selection changed from isolated to shared.

### 4. `with` can use the default profile

```bash
cc-use with
cc-use with -p "review this diff"
cc-use with -- -p "review this diff"
cc-use with deepseek
```

`cc-use with` is now an explicit shared-context spelling for the configured default profile.

`cc-use with auto` remains supported as a compatibility alias for shared auto routing:

```bash
cc-use with auto
cc-use with auto -- -p "review this diff"
```

### 5. `isolate` remains the explicit per-profile session mode

```bash
cc-use isolate
cc-use isolate -p "review this diff"
cc-use isolate -- -p "review this diff"
cc-use isolate deepseek
cc-use isolate auto
cc-use isolate auto -- -p "review this diff"
```

Isolated mode uses:

```text
~/.cc-use/sessions/<profile>/
```

This preserves the previous isolated behavior for experiments, provider-specific history, and compatibility debugging.

## Command semantics

| Command | 0.5.0 behavior |
| --- | --- |
| `cc-use` | default profile, shared native `~/.claude/` |
| `cc-use <profile>` | named profile, shared native `~/.claude/` |
| `cc-use auto` | auto-select usable profile, shared native `~/.claude/` |
| `cc-use with` | default profile, shared native `~/.claude/` |
| `cc-use with <profile>` | named profile, shared native `~/.claude/` |
| `cc-use with auto` | compatibility alias for shared auto |
| `cc-use isolate` | default profile, isolated session dir |
| `cc-use isolate <profile>` | named profile, isolated session dir |
| `cc-use isolate auto` | auto-select usable profile, isolated session dir |

## What did not change

`0.5.0` does not change the auto-routing checker/router abstraction from `0.4.0`.

Unchanged:

- `UsabilityResult`
- checker/router split
- `usable === true` routing rule
- `probe`
- `manual_availability`
- `status.json`
- `cc-use status`

Still not included:

- mid-run provider switching
- provider quality scoring
- automatic model selection
- task-difficulty routing
- real balance adapters
- usage ledger
- session migration between native and isolated modes

## Upgrade notes

This is the key migration:

```text
Since 0.5.0, `cc-use <profile>` uses shared native Claude Code context by default.
Use `cc-use isolate <profile>` for per-profile isolated CLAUDE_CONFIG_DIR.
```

Existing isolated session directories are preserved. `0.5.0` does not delete, rewrite, or migrate:

```text
~/.cc-use/sessions/<profile>/
```

`remove --delete-session` remains the explicit cleanup path. `import-history` remains useful for isolated sessions, but it is no longer required for the default daily-driver path.
