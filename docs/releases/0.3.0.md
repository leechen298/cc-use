# cc-use 0.3.0

> MiMo support, shared-context `with` mode, and explicit isolated sessions

## Summary

`0.3.0` is the first release where `cc-use` clearly supports two distinct launch modes:

- `with`: reuse native Claude Code context under `~/.claude`
- `isolate`: use a separate provider-specific session under `~/.cc-use/sessions/<profile>/`

This release also adds Xiaomi MiMo provider templates and updates the CLI/docs so the daily-use path is easier to understand.

## Highlights

### 1. `cc-use with <profile>`

New shared-context mode:

```bash
cc-use with deepseek
cc-use with mimo
```

What it does:

- reuses native `~/.claude/`
- keeps existing Claude Code history, skills, and projects
- swaps the backend provider only for the current launch

Recommended for daily use when you already rely on Claude Code locally and do not want to split your working context.

### 2. `cc-use isolate <profile>`

New explicit isolated-session mode:

```bash
cc-use isolate deepseek
cc-use isolate mimo
```

What it does:

- uses `~/.cc-use/sessions/<profile>/`
- keeps provider-specific history separate from native Claude Code
- makes comparison, testing, and compatibility debugging easier

This is also the explicit form of the existing isolated behavior.

### 3. Xiaomi MiMo provider templates

Added built-in templates:

- `mimo`
- `mimo-plan`

Current endpoint mapping:

```text
mimo       -> https://api.xiaomimimo.com/anthropic
mimo-plan  -> https://token-plan-cn.xiaomimimo.com/anthropic
```

`mimo` is the public pay-as-you-go Anthropic-compatible endpoint.  
`mimo-plan` is the China Token Plan endpoint.

## Current command semantics

After `0.3.0`, the recommended mental model is:

```bash
cc-use with <profile>
```

- shared mode
- recommended daily-use path

```bash
cc-use isolate <profile>
```

- explicit isolated mode

```bash
cc-use <profile>
```

- compatibility shorthand
- currently equivalent to `isolate`

```bash
cc-use
```

- launches the default profile
- current default-profile runtime behavior is also isolated mode

## Why this is a minor release

This is better treated as `0.3.0` than another `0.2.x` patch because it includes multiple user-visible feature additions:

- a new launch mode (`with`)
- a new explicit command (`isolate`)
- new built-in providers (`mimo`, `mimo-plan`)
- clarified default launch semantics across README and `--help`

## Upgrade notes

No breaking change is introduced for existing isolated workflows:

- `cc-use <profile>` still behaves as isolated mode
- `cc-use` with a configured default profile still behaves as isolated mode

The new shared-context behavior is opt-in through:

```bash
cc-use with <profile>
```
